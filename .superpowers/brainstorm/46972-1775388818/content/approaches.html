技术方案对比

三种架构方向，各有取舍

A

轻量方案（推荐 ⭐）

状态管理: Zustand — 极简，无 boilerplate

数据库: expo-sqlite — Expo 官方支持，SQL 查询灵活

图表: react-native-gifted-charts — 动画流畅，支持环形图+柱状图

导航: Expo Router (文件路由) — 类似 Next.js，开发体验好

UI 组件: 手写组件 + react-native-reanimated 动效

优势

  • 依赖少，包体积小
  • 代码量适中，开发快
  • Zustand 上手零门槛
  • Expo Router 文件路由很直观

劣势

  • 图表定制性中等
B

均衡方案

状态管理: Redux Toolkit — 成熟生态，DevTools 调试方便

数据库: expo-sqlite — 同上

图表: Victory Native — 功能强大，高度可定制

导航: Expo Router

UI 组件: react-native-paper + reanimated

优势

  • Redux 生态成熟，有中间件
  • Victory 图表高度可定制
  • Paper 提供现成 Material 组件

劣势

  • Redux 模板代码多
  • Victory 依赖较重
  • 对简单 APP 来说过度设计
C

重图表方案

状态管理: Zustand

数据库: expo-sqlite

图表: ECharts (wrn-echarts) — 最强图表能力

导航: Expo Router

UI 组件: 手写组件 + reanimated

优势

  • ECharts 图表极其丰富
  • 与 Web 版 ECharts API 一致

劣势

  • 基于 WebView 渲染，性能略差
  • 包体积大
  • 原生交互手感打折

💡 推荐方案 A：对于"极简账单"这种定位清晰的 APP，轻量方案最匹配。Zustand + expo-sqlite + gifted-charts 组合简洁高效，完全满足环形图和柱状图需求，且开发速度最快。
