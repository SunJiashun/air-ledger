技术架构设计

离线优先 + 云同步的分层架构

架构总览
UI 层
📱 Expo Router（文件路由）
🎨 Theme Provider（莫兰迪/深色）
✨ Reanimated 3（动画/手势）
📊 Gifted Charts（图表）
状态层
🐻 Zustand Store
├─ billStore（账单 CRUD）
├─ categoryStore（分类管理）
├─ themeStore（主题切换）
└─ syncStore（同步状态）
▼
数据层
💾 expo-sqlite（本地数据库）
├─ bills 表
├─ categories 表
└─ sync_queue 表（待同步队列）
同步层
☁️ Supabase
├─ Auth（登录/注册）
├─ Database（PostgreSQL）
├─ Realtime（实时同步）
└─ RLS（行级安全策略）
离线优先同步流程
1. 记账
写入本地 SQLite
→
2. 入队
加入 sync_queue
→
3. 检测网络
NetInfo 监听
→
4. 上传
批量同步到 Supabase
→
5. 确认
清除 sync_queue
⚡ 冲突策略：使用 updated_at 时间戳，以最后修改为准（Last Write Wins）。共享账本场景下，每条记录带 user_id，不同用户的记录不冲突。
数据模型
📋 bills 表
id — UUID 主键
amount — 金额 (REAL)
type — 'income' | 'expense'
category_id — 分类 FK
date — 日期
note — 备注（可选）
user_id — 用户 ID
ledger_id — 账本 ID
created_at — 创建时间
updated_at — 修改时间
is_deleted — 软删除标记
🏷️ categories 表
id — UUID 主键
name — 分类名称
icon — emoji 图标
type — 'income' | 'expense'
color — 图表颜色
sort_order — 排序
is_custom — 是否自定义
user_id — 创建者
is_deleted — 软删除标记
📚 ledgers 表（账本）
id — UUID 主键
name — 账本名称
owner_id — 创建者
created_at — 创建时间
👥 ledger_members 表
ledger_id — 账本 FK
user_id — 用户 FK
role — 'owner' | 'member'
joined_at — 加入时间
技术栈总览
框架 Expo SDK 52 + Expo Router v4
语言 TypeScript
状态管理 Zustand
本地数据库 expo-sqlite
云端服务 Supabase（Auth + PostgreSQL + Realtime）
图表 react-native-gifted-charts
动画 react-native-reanimated 3 + Gesture Handler
网络检测 @react-native-community/netinfo
